# Contributing to ancplua-research

Thank you for your interest in contributing to this research repository!

## How to Contribute

### Reporting Issues

- Use the [bug report template](.github/ISSUE_TEMPLATE/bug_report.md) for bugs.
- Use the [feature request template](.github/ISSUE_TEMPLATE/feature_request.md)
  for new ideas.
- Search existing issues before opening a new one to avoid duplicates.

### Submitting Changes

1. Fork the repository.
2. Create a new branch from `main` with a descriptive name
   (e.g., `feature/add-result-analysis`).
3. Make your changes with clear, focused commits.
4. Ensure any Markdown files pass `markdownlint`
   (the CI workflow checks this automatically).
5. Open a pull request against `main` with a clear description of the changes.

### Adding Results

- Place experiment results, artifacts, and outputs in the `results/` directory.
- Include a brief description in the file or alongside it so others understand
  the context.

### Documentation

- Update `docs/README.md` or add new files under `docs/` for methodology,
  findings, or references.
- Keep documentation concise and accurate.

## Code of Conduct

Be respectful and constructive in all interactions. This is a collaborative
research project.
